When CargoWise opens an email document from eDocs in a WiseCloud environment, it writes a temporary file on the WiseCloud server using the email's subject line as the filename. The WiseCloud session profile path (constructed from the WiseCloud username ACG.Helmut.schlapak@wisecloud.zone) adds approximately 50 characters as a prefix, resulting in a total path length of approximately 285 characters. Windows enforces a hard limit of 260 characters for fully qualified file paths.

When this limit is exceeded, the file creation fails silently — no log entry is generated because the failure occurs before the file-open handoff to Outlook. This is why the WiseCloud log shows a successful connection but no file-open event.

The 13-APR-26 file opens correctly because that email has a shorter subject line, keeping the total path within the 260-character limit. The file size of 1.03 MB is not a factor.

Immediate workaround (already working for you):

When you right-click and save the file from eDocs, you control the destination path on your local machine. Saving to a short path such as C:\Temp\ keeps the total path well within the 260-character limit and allows Outlook to open it normally. This will continue to work reliably for any affected email.

Resolution path (requires WiseTech Global Hosting team):

Because your environment is WiseCloud-hosted, the temp path and the WiseCloud server configuration are managed by WiseTech Global — these are not changes your IT team can make locally. Resolving this permanently requires one or both of the following actions by the WiseTech Global Hosting team:

1. Enable Windows Long Path support on the WiseCloud servers: The Win32 long paths Group Policy can be enabled on the WiseCloud RDS server infrastructure to remove the 260-character path limit for all sessions. This is the primary fix.

2. Truncate the temp filename used for EML file opens: CargoWise can be configured to truncate the email subject line when constructing the temp filename, so the full path stays within the 260-character limit regardless of subject length. This has been applied in other eDocs file-handling flows (reference WI00576556).
