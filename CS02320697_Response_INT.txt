Thank you for the screenshots. I reviewed the messages shown for transaction number DELAEOS262700016.

The first message is explaining that there are currently multiple draft invoices with the same transaction number for the same creditor. In this situation, IPP does not automatically discard the duplicates. Posting is blocked until only one draft remains, because if another draft with the same transaction number is posted first, the remaining draft may no longer be valid to post.

The second message is a separate known IPP issue. The error "The Draft Invoice Job Reference cannot be deleted because there is at least one Acc Draft Invoice Charge Line Job Reference referencing it" matches known work item WI01038413. In validated cases, this happened after the document was uploaded and the draft's job-reference matching changed afterward, for example from unmatched to matched, or from matched to unmatched.

This issue is fixed in CargoWise 26.4.15.299, DP Release 2026 Apr 15 patch 299, or later.

What to do next:

If your system is below CargoWise 26.4.15.299, please arrange upgrade to that build or later. After the upgrade, reopen the three drafts for DELAEOS262700016, discard the two duplicates, then post the remaining correct draft.

If you need a temporary workaround before upgrade, please keep the correct draft and try to reverse the last job-reference change made after upload on the duplicate drafts, then discard those duplicates. Once only one draft remains for DELAEOS262700016, the posting validation should clear.
