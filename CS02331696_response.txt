When CargoWise opens an .eml file in a WiseCloud environment, it transfers the file through the Remote Desktop Services file redirect channel and writes it as a temporary file on the local machine before handing it to Outlook. There are two document-level factors that can cause this transfer to fail silently and produce the RDS interruption error:

1. File size — If the failing email contains large embedded attachments or inline images, the RDS file redirect may time out or be interrupted before the write completes. The working file likely does not have this volume of embedded content.

2. File name length — CargoWise uses the email subject line as the document file name. If the subject line is very long, the resulting Windows temp path (temp folder path + subject + .eml extension) can exceed the 260-character Windows path limit, causing the temporary file creation to fail before Outlook is invoked.

The WiseCloud log not recording a file-open event for the failing document is consistent with both of these failure modes — the operation stops before it is logged.
